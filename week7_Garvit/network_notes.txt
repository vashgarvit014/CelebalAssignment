Computer Networks Notes

Chapter 1: Introduction to Networking
A computer network is a collection of interconnected devices that can exchange data and share resources with each other. Networks are typically classified by their geographic scope. A LAN, or local area network, covers a small area such as a single building or campus. A WAN, or wide area network, spans large geographic distances and often connects multiple LANs together, with the internet being the largest example of a WAN. A MAN, or metropolitan area network, falls in between, typically covering a city.

Chapter 2: The OSI Model
The OSI model, or Open Systems Interconnection model, divides network communication into seven distinct layers, each responsible for a specific set of functions. From bottom to top, these layers are the physical layer, data link layer, network layer, transport layer, session layer, presentation layer, and application layer. The physical layer deals with the actual transmission of raw bits over a physical medium such as cables or radio waves. The data link layer handles node to node data transfer and error detection within the same network segment. The network layer is responsible for logical addressing and routing packets across different networks. The transport layer ensures reliable end to end data delivery, with TCP and UDP being the two primary protocols at this layer. The session layer manages sessions between applications. The presentation layer handles data formatting, encryption and compression. The application layer is closest to the end user and includes protocols like HTTP and FTP.

Chapter 3: The TCP/IP Model
The TCP/IP model is a more practical four layer model that is actually used on the internet, consisting of the network access layer, internet layer, transport layer and application layer. Unlike the OSI model which is largely theoretical, TCP/IP was designed based on how the internet actually operates. The internet layer, primarily handled by the IP protocol, is responsible for addressing and routing packets between networks. IP addresses uniquely identify devices on a network. IPv4 addresses are 32 bits long and are typically written in dotted decimal notation, while IPv6 addresses are 128 bits long and were introduced to solve the problem of IPv4 address exhaustion.

Chapter 4: Transport Layer Protocols
TCP, or Transmission Control Protocol, is a connection oriented protocol that guarantees reliable, ordered delivery of data through mechanisms like acknowledgments, retransmission and sequence numbers. Before data transfer begins, TCP establishes a connection using a three way handshake, where the client sends a SYN packet, the server responds with a SYN ACK packet, and the client finally sends an ACK packet to complete the handshake. UDP, or User Datagram Protocol, is connectionless and does not guarantee delivery, ordering or error checking, which makes it faster and suitable for applications like video streaming and online gaming where speed matters more than perfect reliability.

Chapter 5: IP Addressing and Subnetting
Subnetting is the process of dividing a larger network into smaller, more manageable sub networks called subnets. This is done using a subnet mask, which determines which portion of an IP address represents the network and which portion represents the host. CIDR notation, written as an IP address followed by a slash and a number such as 192.168.1.0/24, indicates how many bits are used for the network portion. Subnetting improves network performance by reducing broadcast traffic and enhances security by isolating different segments of a network from each other.

Chapter 6: Routing Protocols
Routing is the process of selecting a path for traffic to travel across one or more networks. Distance vector routing protocols, such as RIP, determine the best path based on the number of hops to a destination, with each router sharing its entire routing table with its neighbors periodically. Link state routing protocols, such as OSPF, build a complete map of the network topology by having each router share information about its directly connected links, which allows for faster convergence and more accurate path selection compared to distance vector protocols. BGP, or Border Gateway Protocol, is the routing protocol that governs how data is routed between different autonomous systems on the internet.

Chapter 7: Congestion Control
Congestion control refers to the techniques used to prevent a network from becoming overwhelmed with too much traffic. TCP implements congestion control through mechanisms like slow start, where the sender begins by transmitting a small amount of data and gradually increases the transmission rate as long as no packet loss is detected. Congestion avoidance kicks in once a threshold is reached, increasing the transmission rate more conservatively. When packet loss is detected, TCP reduces its transmission rate significantly through a mechanism called congestion window reduction, then attempts to recover gradually. These mechanisms work together to ensure fair and efficient use of network bandwidth among competing connections.

Chapter 8: Application Layer Protocols
DNS, or Domain Name System, translates human readable domain names into IP addresses that computers use to identify each other on the network, functioning much like a phonebook for the internet. HTTP, or Hypertext Transfer Protocol, is the foundation of data communication on the web, defining how messages are formatted and transmitted between web browsers and servers. HTTPS adds a layer of encryption to HTTP using TLS, protecting data from being intercepted or tampered with during transmission. FTP, or File Transfer Protocol, is used for transferring files between a client and a server over a network.

Chapter 9: Network Security Basics
A firewall is a network security device that monitors incoming and outgoing traffic and decides whether to allow or block specific traffic based on a defined set of security rules. Firewalls can be hardware based, software based, or a combination of both. Encryption is the process of converting data into a coded format that can only be read by someone with the correct decryption key, protecting sensitive information from unauthorized access during transmission. A VPN, or Virtual Private Network, creates an encrypted tunnel between a device and a network, allowing secure remote access and protecting data from being intercepted on untrusted networks such as public wifi.

Chapter 10: Switching Techniques
Circuit switching establishes a dedicated communication path between two devices for the entire duration of a session, similar to how traditional telephone networks operated, guaranteeing consistent bandwidth but potentially wasting resources during idle periods. Packet switching, which is what modern computer networks including the internet use, breaks data into smaller packets that are routed independently and can take different paths to reach the same destination, making more efficient use of available bandwidth. Each packet contains header information including source and destination addresses that allow routers to forward it correctly.
